# 🚀 グループToDo管理アプリ (Student Edition)

このアプリは、グループ活動でのイベントまでの残り日数や、タスク（ToDo）の進捗状況を「見える化」するためのシンプルなWebアプリです。

## 🌟 このアプリの特徴
- **GitHubがデータベース**: プログラムを書き換えなくても、`json` ファイルを編集するだけで画面が更新されます。
- **スマホ対応**: 外出先でも進捗を確認しやすいシンプルなデザインです。
- **AI連携の練習**: 改善案をメモして、AIと一緒にアプリを育てるきっかけになります。

---

## 🛠 使い方（学生のみなさんへ）

自分のグループの進捗を表示・更新するには、以下の手順でファイルを編集してください。

### 1. 自分のグループフォルダを探す
`groups/groupXX/`（XXは自分の班番号）の中に、あなたたちのデータが入っています。

### 2. 基本情報を設定する (`config.json`)
グループ名や目標とするイベントの日程を編集します。
- `groupName`: 班の名前
- `eventName`: 目標とするイベント（例：「中間発表」「夏合宿」）
- `eventDate`: イベントの日（YYYY-MM-DD 形式）

### 3. タスクを追加・更新する (`tasks.json`)
ToDoリストの中身です。以下のような形式で書き込みます。

```json
[
  {
    "title": "ここにタスク名",
    "deadline": "2026-06-20",
    "status": "未着手",
    "owner": "担当者名"
  }
]
```
- **status**: `未着手`, `作業中`, `完了` の3つから選んでください（これ以外だと色がつきません）。

### 4. 改善案をメモする (`improvement_ideas.md`)
「こんな機能があったらいいな」と思ったら、このファイルに書き残してください。後の演習でAIを使って実際に機能を追加する際のヒントになります。

---

## ⚠️ 注意点（JSONの書き方）
JSONファイルを編集するときは、以下のルールを守ってください。間違えると画面が真っ白になってしまいます！
1. **半角**で入力する（全角のスペースやカンマはエラーになります）。
2. 文字列は **ダブルクォーテーション `"`** で囲む。
3. 項目と項目の間は **カンマ `,`** で区切る。
4. **一番最後の項目の後ろにはカンマを付けない。**

## 📂 フォルダ構成
```text
.
├── app/                 # アプリ本体（基本は触りません）
│   ├── index.html       # 画面の見た目
│   ├── style.css        # デザイン
│   └── script.js        # 動き（データの読み込み）
├── groups/              # 【重要】みんなが編集する場所
│   ├── group01/
│   │   ├── config.json  # グループの設定
│   │   ├── tasks.json   # タスクの一覧
│   │   └── improvement_ideas.md # 改善アイディア
│   └── ...
└── README.md            # この説明書
```

### GitHubへアップする際のヒント
1. **リポジトリ名**: GitHub Pagesで公開した際、URLの一部になります。
2. **公開設定**: `Settings` > `Pages` から、`Branch: main` / `Folder: / (root)` を選択して保存してください。
3. **リンクの案内**: 公開されたURLを学生に共有する際、末尾に `/app/index.html` をつけるのを忘れないように伝えてください。
