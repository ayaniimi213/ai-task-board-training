# 🚀 グループToDo管理アプリ (Student Edition)

このアプリは、グループ活動でのイベントまでの残り日数や、タスク（ToDo）の進捗状況を「見える化」するためのシンプルなWebアプリです。

## 🌟 このアプリの特徴
- **GitHubがデータベース**: プログラムを書き換えなくても、`json` ファイルを編集するだけで画面が更新されます。
- **スマホ対応**: 外出先でも進捗を確認しやすいシンプルなデザインです。
- **AI連携の練習**: 改善案をメモして、AIと一緒にアプリを育てるきっかけになります。

---

## 🛠 準備：グループフォルダの一括作成（管理者・教員用）

演習を開始する前に、管理者の方は以下の手順で各グループ（01〜20）のフォルダを作成してください。

1.  Pythonがインストールされている環境で、リポジトリのルート（このREADMEがある場所）に `create_groups.py` を配置します。
2.  以下のコマンドを実行します。
    ```bash
    python create_groups.py
    ```
3.  `groups/group01/` 〜 `groups/group20/` までのフォルダと、初期データ（`config.json`, `tasks.json`, `improvement_ideas.md`）が自動生成されます。
4.  生成されたファイルを GitHub に `push` してください。

---

## 📖 使い方（学生のみなさんへ）

自分のグループの進捗を表示・更新するには、以下の手順でファイルを編集してください。

### 1. 自分のグループフォルダを探す
`groups/groupXX/`（XXは自分の班番号）の中に、あなたたちのデータが入っています。

### 2. 基本情報を設定する (`config.json`)
グループ名や目標とするイベントの日程を編集します。
- `groupName`: 班の名前（例：「チーム・フロントエンド」）
- `eventName`: 目標とするイベント（例：「中間発表」「プロトタイプ完成」）
- `eventDate`: イベントの日（YYYY-MM-DD 形式。例：`"2026-08-01"`）

### 3. タスクを追加・更新する (`tasks.json`)
ToDoリストの中身です。以下のような形式で書き込みます。

```json
[
  {
    "title": "ここにタスク名",
    "deadline": "2026-06-20",
    "status": "未着手",
    "owner": "担当者名"
  }
]
```
- **status**: `未着手`, `作業中`, `完了` の3つから選んでください。

### 4. 改善案をメモする (`improvement_ideas.md`)
「こんな機能があったらいいな」と思ったら、このファイルに書き残してください。

---

## ⚠️ 注意点（JSONの書き方）
JSONファイルを編集するときは、以下のルールを守ってください。**間違えると画面に何も表示されなくなります！**
1. **半角**で入力する（全角のスペースやカンマはエラーになります）。
2. 文字列は **ダブルクォーテーション `"`** で囲む。
3. 項目と項目の間は **カンマ `,`** で区切る。
4. **一番最後の項目の後ろにはカンマを付けない。**
